## 明日计划

### 1. 检查新版 reasoning 数据质量

- 随机抽样 50-100 条新版 reasoning 数据进行人工检查。
- 重点检查以下内容：
  - 题型判断是否正确；
  - reasoning 是否真正有助于解题；
  - 最终答案是否与标准答案一致；
  - reasoning 是否存在模板化、废话过多或错误推理；
  - 输出格式是否稳定、可解析。
- 将发现的问题分类记录：
  - 题型判断错误；
  - 推理过程错误；
  - 最终答案错误；
  - 格式不规范；
  - reasoning 过长或无效。

### 2. 用新版 reasoning 数据做小规模测试

- 先抽取小规模数据进行 sanity check。
  - 训练集：100-200 条
  - 验证集：20-50 条
- 目标是确认模型是否能在小数据上明显 overfit。
- 如果小数据仍然学不动，优先排查：
  - prompt 和 response 拼接格式；
  - label masking 是否正确；
  - learning rate 是否合适；
  - reasoning 与 final answer 的格式是否统一；
  - 训练数据字段是否存在错位。

### 3. 重新训练 500 条新版 reasoning 数据

- 在小规模 sanity check 正常后，再使用新版 500 条 reasoning 数据重新训练 LoRA。
- 保持当前 450/50 的 train/test 分配，方便与上一轮结果对比。
- 记录训练配置：
  - base model；
  - LoRA 参数；
  - learning rate；
  - batch size；
  - epoch；
  - max length；
  - prompt 模板版本；
  - reasoning 数据版本。

### 4. 固定评估方式并做对比实验

- 固定 50 条 test set 作为评估集。
- 至少对比以下实验：
  - 原模型 baseline；
  - 只训练最终答案，不加入 reasoning；
  - 训练 reasoning + final answer；
  - 新版 reasoning 标注训练结果。
- 评估指标：
  - overall accuracy；
  - exact match；
  - 按题型统计 accuracy；
  - final answer 是否可解析；
  - reasoning 与 final answer 是否一致。

### 5. 分析错误样本

- 对测试集中错误样本进行逐条分析。
- 按错误类型归因：
  - 题型识别错误；
  - 推理链错误；
  - 最终答案抽取错误；
  - 模型输出格式错误；
  - 数据标注本身错误。
- 根据错误类型决定下一步是继续修数据、改 prompt，还是调整训练参数。

### 6. 初步设计奖励机制

- 暂时先设计 rule-based reward，不急于直接大规模上 GRPO。
- 奖励项可以包括：
  - 答案正确性 reward；
  - 输出格式 reward；
  - 题型判断 reward；
  - reasoning 与 final answer 一致性 reward；
  - reasoning 过长惩罚。
- 初步 reward 结构：

```text
total_reward =
  0.7 * answer_correct
+ 0.1 * format_valid
+ 0.1 * question_type_correct
+ 0.1 * reasoning_answer_consistent
- 0.05 * excessive_length_penalty
```

### 7. 准备引入 GRPO

- 在新版 reasoning SFT 能稳定提升后，再引入 GRPO。
- GRPO 前需要先准备：
  - 稳定可解析的模型输出格式；
  - 固定评估集；
  - rule-based reward 函数；
  - 可复现实验配置；
  - baseline 结果记录。

3. 确认数据和训练流程没有明显问题后，再跑完整 500 条 LoRA。
